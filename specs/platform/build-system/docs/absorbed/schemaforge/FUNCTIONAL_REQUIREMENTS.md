# Functional Requirements: craft

## FR-CRT-001: Schema Format
FR-CRT-001a: craft SHALL accept schema files in YAML and JSON formats.
FR-CRT-001b: Each entity SHALL define a name, fields (with type and constraints), and optional relationships.
FR-CRT-001c: Supported field types SHALL include: string, int, float, bool, date, datetime, uuid, bytes, and arrays/maps of the above.
FR-CRT-001d: craft SHALL support enum definitions with named variants.
FR-CRT-001e: Schemas SHALL support `$extends` to inherit fields from a base entity.

## FR-CRT-002: CLI Commands
FR-CRT-002a: `craft generate --schema <file> --output <dir> --lang <langs>` SHALL generate code into the output directory.
FR-CRT-002b: `craft watch --schema <file> --output <dir>` SHALL watch the schema file and regenerate on change.
FR-CRT-002c: `craft validate --schema <file>` SHALL validate schema syntax and type references, printing all errors.
FR-CRT-002d: `craft diff --schema <file> --output <dir>` SHALL print what files would be created/modified/deleted without writing.

## FR-CRT-003: Rust Generator
FR-CRT-003a: The Rust generator SHALL emit a struct per entity with `#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]`.
FR-CRT-003b: The Rust generator SHALL emit an enum per enum definition with serde rename rules.
FR-CRT-003c: Optional fields SHALL be rendered as `Option<T>`.
FR-CRT-003d: The Rust generator SHALL emit a builder pattern for entities with more than 3 fields.

## FR-CRT-004: TypeScript Generator
FR-CRT-004a: The TypeScript generator SHALL emit a `type` alias per entity using the `model` library schema.
FR-CRT-004b: The TypeScript generator SHALL emit enum types as TypeScript `const enum` or union string literals.
FR-CRT-004c: Generated files SHALL include a `// @generated` header comment.

## FR-CRT-005: Go Generator
FR-CRT-005a: The Go generator SHALL emit a struct per entity with `json:` tags.
FR-CRT-005b: Optional fields SHALL be rendered as pointer types (`*T`).
FR-CRT-005c: Generated files SHALL include a `// Code generated by craft. DO NOT EDIT.` header.

## FR-CRT-006: Template Overrides
FR-CRT-006a: craft SHALL load custom templates from `.craft/templates/<lang>/` if present in the project directory.
FR-CRT-006b: Custom templates SHALL override built-in templates for matching generator/entity type combinations.

## FR-CRT-007: Output Management
FR-CRT-007a: craft SHALL write a `.craft-manifest.json` listing all generated files and their source entity.
FR-CRT-007b: craft SHALL delete generated files that are no longer referenced by the current schema (stale cleanup).
